# Part I — Codebase Analysis

_Generated 2026-05-28 via a multi-agent analysis workflow (per-subsystem deep-dive + cross-cutting investigations, synthesized)._

## 1. Overview

Papyri decouples two things Sphinx fuses together: producing documentation and rendering it. The system is a four-stage pipeline with one IR (Intermediate Representation) flowing through it:

1. **Generate** (`papyri gen`, Python) — introspects a library, parses docstrings/RST via tree-sitter, runs doctests, and emits a human-readable JSON DocBundle under `~/.papyri/data/<pkg>_<ver>/`.
2. **Pack** (`papyri pack`, Python) — validates the bundle and encodes it into a single deterministic, gzip-compressed CBOR `.papyri` artifact. CBOR begins here, never before.
3. **Ingest** (`papyri-ingest`, TypeScript) — decodes the artifact, walks the IR tree, computes content digests, and writes nodes + cross-link edges into a SQLite graph DB and a filesystem blob store. Invoked server-side by the viewer's `PUT /api/bundle`. The raw `.papyri.gz` is archived under `_raw/` as the only authoritative copy; the graph is a rebuildable cache.
4. **Render** (`viewer/`, Astro SSR + TypeScript) — decodes blobs back into typed IR, resolves cross-references against the graph (with intersphinx fallback for non-papyri projects), and serves multi-version API/narrative/example pages.

**The integrating insight: the IR is a contract spanning Python and TypeScript, and almost nothing automated checks that the two sides agree.** The most serious structural risks all cluster on this seam — CBOR field order is kept in sync by hand across languages, the digest scheme cannot distinguish a real content change from a schema bump, the Python and TS node-type registries have drifted apart, and no test exercises the full gen→pack→ingest→render path. Each subsystem is individually well-built; the fragility lives in the joints between them.

## 2. Architecture map

**Python IR producer + CLI (`papyri/`)** — A multi-stage pipeline: `DFSCollector` walks imported module objects to build an inventory; per-object extraction parses docstrings with the tree-sitter RST wrapper; `GenVisitor`/`DirectiveVisiter` traverse parsed sections for reference resolution and directive dispatch; nodes serialize to JSON, then `pack` validates layout and encodes canonical CBOR. Figures dedup by content hash and intra-bundle refs use version-less `LocalRef`, keeping the digest independent of the bundle's own version. Key files: `gen.py`, `ts.py`, `tree.py`, `nodes.py`/`node_base.py`/`node_serializer.py`, `pack.py`, `cli/*.py`.

**TypeScript ingest engine (`ingest/`)** — Three-phase: bulk pre-fetch of existing bundle state, a CPU-only accumulation phase that walks the IR, re-encodes docs to CBOR, collects forward refs and stages blob+graph statements, then a flush phase with concurrent blob writes and chunked DB transactions (node statements ordered before link statements so FKs resolve). Pluggable behind `BlobStore`/`GraphDb`/`RawStore`; only fs + SQLite implementations exist. Key files: `ingest.ts`, `encoder.ts`, `visitor.ts`, `bundle.ts`, `keys.ts`, `inventory.ts`, `migrations/*.sql`.

**Viewer rendering library (`viewer/src/lib/`)** — The shock-absorber between CBOR and Astro templates. `ir-reader.ts` decodes blobs into typed IR; `graph.ts` wraps the cross-link DB; `xref.ts` batches ref resolution; `render-node.ts` dispatches 50+ node types to HTML; `qualname-page.ts`/`doc-page.ts` build view models; `math.ts`/`highlight.ts` pre-render KaTeX/Shiki server-side.

**Viewer Astro app (`viewer/src/pages|components|layouts`)** — Full SSR app over the lib layer. Upload path: gunzip → CBOR decode → `Ingester.ingestBundle()` → streaming NDJSON progress. Read path: memoized `getBackends()` → per-bundle `loadBundleNav()` → `renderNode()`. Auth middleware enforces `PUBLIC_PREFIXES` / `ADMIN_ONLY_PREFIXES`.

## 3. Strengths

- **Deterministic, content-addressed packing.** Canonical CBOR + gzip → byte-reproducible artifacts; `pack` refuses to ship a bundle whose `papyri.json` records gen errors. Strongest test suite in the repo (`test_pack.py`).
- **Digest churn control on both sides** — gen strips `lineno`/uses `LocalRef`; ingest's `stripVolatileFields()` nulls volatile fields before hashing.
- **Type-safe tagged-union IR** with directly-tested round-trip fidelity (`test_cbor_roundtrip.py`, `encoder.test.ts`).
- **Clean layering / visitor reuse**, batched ref resolution avoids N+1.
- **Defense in depth on untrusted input** — `safeJoin()` blocks traversal, `isSafeUrl()` rejects `javascript:`/`data:` at both ingest and render time.
- **Rebuildable-cache discipline** — `_raw/` authoritative, `POST /api/reingest` replays.
- **Strong test depth where it exists** — RST→IR (`test_tree.py`), intersphinx, render-node HTML.

## 12. Verified-correct (checked and cleared)

The substance of the subsystem is sound:

- **No SQL injection** — every query is parameterized, including all joins.
- **Account/token endpoints are IDOR-safe** — `api/account/password.ts` and `api/account/tokens.ts` derive the user from the session (`resolveSession`), never a body `userId`; token revoke is scoped `WHERE id = ? AND user_id = ?`. Both also **re-check the session in-handler** (the backstop A1 wants for the admin routes).
- **Password change is safe** — verifies the current password first, then revokes the user's *other* sessions so a leaked session can't keep riding old creds.
- **Upload auth (`api/bundle.ts`) is fail-closed and correct** — bearer→global token compared with `timingSafeEqual`; else resolved as a personal token; no-bearer is `open` **only** on a zero-user fresh install, else 401. Project scope is authorized against the *decoded* `bundle.module` after an `isSafeSegment` path-traversal guard, and a project-scoped token is enforced even for admins.
- **Lockout protection** — `setAdmin` and `users` DELETE both refuse to remove the last admin (and DELETE the last user); session/membership/token rows cascade on user delete (`foreign_keys = ON`).
- **Token/session secrets** — sessions are 256-bit random; personal upload tokens are 192-bit random stored as SHA-256 (a fast hash is correct for a high-entropy secret), shown once.
- **Route gating coverage** — every destructive/admin route *is* listed in `ADMIN_ONLY_PREFIXES`, and the `prefix`-boundary match correctly distinguishes `/api/clear` from `/api/clear-raw` and catches `/api/projects/members`. (The A1 gap is about *evading* the classifier, not a missing entry.)
