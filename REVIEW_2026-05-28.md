# Papyri Codebase Analysis Report

_Generated 2026-05-28 via a multi-agent analysis workflow (per-subsystem deep-dive + cross-cutting investigations, synthesized)._

## 1. Overview

Papyri decouples two things Sphinx fuses together: producing documentation and rendering it. The system is a four-stage pipeline with one IR (Intermediate Representation) flowing through it:

1. **Generate** (`papyri gen`, Python) — introspects a library, parses docstrings/RST via tree-sitter, runs doctests, and emits a human-readable JSON DocBundle under `~/.papyri/data/<pkg>_<ver>/`.
2. **Pack** (`papyri pack`, Python) — validates the bundle and encodes it into a single deterministic, gzip-compressed CBOR `.papyri` artifact. CBOR begins here, never before.
3. **Ingest** (`papyri-ingest`, TypeScript) — decodes the artifact, walks the IR tree, computes content digests, and writes nodes + cross-link edges into a SQLite graph DB and a filesystem blob store. Invoked server-side by the viewer's `PUT /api/bundle`. The raw `.papyri.gz` is archived under `_raw/` as the only authoritative copy; the graph is a rebuildable cache.
4. **Render** (`viewer/`, Astro SSR + TypeScript) — decodes blobs back into typed IR, resolves cross-references against the graph (with intersphinx fallback for non-papyri projects), and serves multi-version API/narrative/example pages.

**The integrating insight: the IR is a contract spanning Python and TypeScript, and almost nothing automated checks that the two sides agree.** The most serious structural risks all cluster on this seam — CBOR field order is kept in sync by hand across languages, the digest scheme cannot distinguish a real content change from a schema bump, the Python and TS node-type registries have drifted apart, and no test exercises the full gen→pack→ingest→render path. Each subsystem is individually well-built; the fragility lives in the joints between them.

## 2. Architecture map

**Python IR producer + CLI (`papyri/`)** — A multi-stage pipeline: `DFSCollector` walks imported module objects to build an inventory; per-object extraction parses docstrings with the tree-sitter RST wrapper; `GenVisitor`/`DirectiveVisiter` traverse parsed sections for reference resolution and directive dispatch; nodes serialize to JSON, then `pack` validates layout and encodes canonical CBOR. Figures dedup by content hash and intra-bundle refs use version-less `LocalRef`, keeping the digest independent of the bundle's own version. Key files: `gen.py` (collection, extraction, doctest execution), `ts.py` (tree-sitter RST parse), `tree.py` (visitor + directive handlers), `nodes.py`/`node_base.py`/`node_serializer.py` (IR schema + CBOR), `pack.py` (artifact encode + validation), `cli/*.py`.

**TypeScript ingest engine (`ingest/`)** — Three-phase: bulk pre-fetch of existing bundle state (avoids N per-doc lookups), a CPU-only accumulation phase that walks the IR, re-encodes docs to CBOR, collects forward refs and stages blob+graph statements, then a flush phase with concurrent blob writes and chunked DB transactions (node statements ordered before link statements so FKs resolve). Pluggable behind `BlobStore`/`GraphDb`/`RawStore`; only fs + SQLite implementations exist. Key files: `ingest.ts` (orchestrator), `encoder.ts` (CBOR + `GeneratedDoc`→`IngestedDoc` field reorder), `visitor.ts` (forward-ref collection), `bundle.ts` (validation), `keys.ts`, `fs-safe.ts`, `url-safety.ts`, `inventory.ts`, `migrations/*.sql`.

**Viewer rendering library (`viewer/src/lib/`)** — The shock-absorber between CBOR and Astro templates. `ir-reader.ts` decodes blobs into typed IR (the designated landing spot for IR changes); `graph.ts` wraps the cross-link DB (exact + version-fallback resolution); `xref.ts` batches ref resolution into a sync closure for use inside render trees; `render-node.ts` dispatches 50+ node types to HTML with graceful fallback; `qualname-page.ts`/`doc-page.ts` build view models; `math.ts`/`highlight.ts` pre-render KaTeX/Shiki server-side. Key files as listed plus `links.ts`, `bundle-walk.ts`, `backends.ts`.

**Viewer Astro app (`viewer/src/pages|components|layouts`)** — Full SSR app over the lib layer. Upload path: gunzip → CBOR decode → `Ingester.ingestBundle()` → streaming NDJSON progress. Read path: memoized `getBackends()` → per-bundle `loadBundleNav()` → `renderNode()`. Auth middleware enforces `PUBLIC_PREFIXES` / `ADMIN_ONLY_PREFIXES` (session cookie) and bearer token on upload. Key files: `pages/api/bundle.ts`, `pages/api/reingest.ts`, `middleware.ts`, `layouts/BundleLayout.astro`, `pages/project/[pkg]/[ver]/[...slug].astro`, `pages/admin/index.astro`, `pages/api/auth/login.ts`.

## 3. Strengths

- **Deterministic, content-addressed packing.** Canonical CBOR (RFC 8949) + gzip gives byte-reproducible artifacts; `pack` refuses to ship a bundle whose `papyri.json` records gen errors, so degraded output never escapes silently (`pack.py`). Confirmed by the strongest test suite in the repo (`test_pack.py`, 941 lines, 60+ cases).
- **Digest churn control on both sides.** Gen strips `lineno` from figure names and anchors intra-bundle refs with version-less `LocalRef`; ingest's `stripVolatileFields()` nulls `Figure.value`/`Image.url`/`item_file`/`item_line` before hashing — so identical content across rebuilds collapses to one blob and doesn't trigger false diffs.
- **Type-safe tagged-union IR.** Internally-tagged CBOR encoding (`node_serializer.py`) preserves exact union types on round-trip, and the TS encoder mirrors Python's `typing.get_type_hints()` order. Round-trip fidelity is directly tested (`test_cbor_roundtrip.py`, `ingest/tests/encoder.test.ts`).
- **Clean layering and visitor reuse.** Parsing, semantic traversal, and rendering each use a visitor with registry-based dispatch rather than monoliths. The viewer keeps IR decode, graph queries, and rendering in separate modules, and batches ref resolution to avoid N+1.
- **Defense in depth on untrusted input.** `safeJoin()` blocks path traversal and `isSafeUrl()` rejects `javascript:`/`data:` URLs at *both* ingest time (`assertSafeUrls`) and render time — the single allowlist (`url-safety.ts`) is shared.
- **Rebuildable-cache discipline.** The graphstore is explicitly derived; `_raw/<pkg>/<ver>.papyri.gz` is authoritative and `POST /api/reingest` replays it, enabling schema evolution without re-uploading from clients.
- **Strong test depth where it exists.** RST→IR transformation (`test_tree.py` 2026 lines / `test_parse.py` 916 lines, 200+ cases), external/intersphinx resolution (`external.test.ts`), and render-node HTML output (`render-node.test.ts`) are all comprehensively covered.

## 5. Recommendations

Prioritized, and scoped to the project's rules (delete dead code rather than keep compat shims; no Python-side rendering; small focused changes).

**P0 — security and contract correctness (each a small, self-contained PR):**

1. **Fix the auth endpoint** (`login.ts`): fail closed when `PAPYRI_USERNAME`/`PAPYRI_PASSWORD` are unset (refuse login + warn), and make the session token unforgeable — sign it (HMAC over user+expiry) and validate the expiry on read. Single highest-value change.
2. **Guard the CBOR field-order contract.** Add a unit test (and a CI gate) asserting `generatedDocToIngested()`'s returned key order equals `FIELD_ORDER[4010]`, so a future reorder fails loudly instead of corrupting the wire format (`encoder.ts:46-68, 301-322`).
3. **Pin Python dependencies.** Add lower+upper bounds in `pyproject.toml` (especially `ipython`, `jedi`, `matplotlib`), pin `requirements-dev.txt`, and pin ruff/mypy in `.pre-commit-config.yaml` to match CI.
4. **Fix the version sort** in `graph.ts:66`: replace `localeCompare` with the existing `compareVersionsDesc` (`ir-reader.ts:33`). One-line, fixes the `2.0.0`-vs-`10.0.0` bug; track the silent-fallback diagnostic (facet b) as a separate design follow-up.
5. **Redirect the `papyri ingest` help text** (`describe.py:85`) to `papyri upload` / the viewer endpoint — do **not** re-add the CLI.

**P1 — close the contract test gap and harden error handling:**

6. **Add one end-to-end pipeline test** (gen a tiny real package → pack → ingest into a temp graph → render a page), the single most valuable test addition; it's the only thing that would catch a Python↔TS schema disagreement. Then backfill direct unit tests for `nodes.py`, `serde.py`, and `ts.py` parse-error paths.
7. **Replace assertion-based validation** in `node_base.py:53` with a raised `TypeError` (note `python -O` disables asserts), and fix the inconsistent `_invalidate` dict-branch error strings (`:195`).
8. **Preserve `Key` objects** in `visitor.ts` instead of `/`-joining/splitting, and add a completeness assertion to forward-ref collection so unrecognized node types fail instead of silently dropping edges.
9. **Add a request mutex** around `reingest`/`clear` admin endpoints.
10. **Turn the worst silent-degradation paths into surfaced errors/warnings** per the project's "prefer errors over text fallback" convention — `bundle-walk.ts` load failures and Jedi inference failures should at least log.

**P2 — quick wins, bundled (one "delete dead code" PR, per CLAUDE.md rule 0):**

11. Delete, don't keep for compat: `processed_example_data` no-op (`gen.py:184-192`), the `IntermediateNode`/`Leaf` node types and the dormant `IngestedDoc` CBOR tag 4010 in `crosslink.py`, the dead non-array `sectionChildren` guard (`ir-reader.ts:169-171`), the stale "BFS collector" comment (`gen.py:354`), and consolidate the duplicated `escapeHtml` (`render-node.ts:23-28`, `highlight.ts:34-36`) into one shared helper.

**P3 — design follow-ups (track in PLAN.md, not quick fixes):**

12. A schema-version-aware content-identity digest that survives IR field additions (concern #7); a diagnostic field recording when a ref was version-fallback-resolved (concern #6b); and aligning mypy/ruff target versions to the supported floor and unifying the pnpm version between CI and Docker (concern #14).

Key files for the P0/P1 work: `viewer/src/pages/api/auth/login.ts`, `ingest/src/encoder.ts`, `viewer/src/lib/graph.ts`, `papyri/cli/describe.py`, `pyproject.toml`, `papyri/node_base.py`, `ingest/src/visitor.ts`.
